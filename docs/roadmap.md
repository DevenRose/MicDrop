# Roadmap

MicDrop starts as a do-one-thing-well Windows tool. Where it goes next is driven
by **what users actually need** — not a guess.

## Now (v1)

- ✅ Stereo-lock toggle (CLI + tray) for any Bluetooth headset on Windows
- ✅ Auto-start, always-visible tray icon
- ✅ Emulated tests per headset combo
- ✅ Beta combo-report funnel

## Next — driven by you

Once MicDrop has solved your headset problem, we want to know **what else is
broken** in your setup. Two ways in:

- The tray menu → **"Got another tech problem? Tell us"** (opens Discussions)
- [GitHub Discussions](https://github.com/OWNER/MicDrop/discussions) directly

Common candidates we may expand into (the "room for more" path):

- Other top-of-search Windows audio gremlins (default-device flapping,
  exclusive-mode conflicts, monitor/USB-C display-audio stealing default)
- A small general "fix-it" toolkit if there's demand

## Beta program

Real-hardware reports beat assumptions. If you try MicDrop with a headset that's
only got an 🧪 emulated test in [headsets.md](headsets.md), a one-line
[combo report](../.github/ISSUE_TEMPLATE/combo-report.md) graduates it to ✅ —
and surfaces any quirks we should fix.

## Support the project

MicDrop is free and stays free. If it saved you an afternoon of fighting Windows
audio, tips are welcome but never required — see the **Sponsor** button or
[FUNDING.yml](../.github/FUNDING.yml).
